Precision Temperature Profiling System

Designed and deployed a custom thermocouple array to map thermal gradients across a 12-zone industrial oven, reducing product defect rates by 18% through improved process control.

Goal:The existing single-point measurement couldn't capture zone-to-zone variation, leading to inconsistent curing and scrap. We needed spatial resolution to identify hot spots and optimize conveyor speed.

Approach:Installed a 24-channel K-type thermocouple grid with a custom DAQ running at 10 Hz. Wrote Python scripts to visualize real-time heatmaps and export CSV logs for statistical analysis. Calibrated against NIST-traceable standards to ±0.5°C.

Validation:Ran three production batches with continuous logging. Identified a 15°C delta between zones 7 and 8. Adjusted heater PID parameters and re-validated. Final variance dropped to ±3°C across all zones, meeting spec.

Initial prototype used off-the-shelf terminal blocks, but vibration caused intermittent connections. Switched to screw-clamp industrial connectors with strain relief. Shielded twisted-pair cables reduced EMI noise from adjacent motor drives.

Data acquisition ran on a Raspberry Pi 4 with a custom HAT board. Chose SPI interface for the ADC to minimize latency. Implemented a rolling buffer to handle occasional network dropouts without losing samples.

Sampling rate: 10 Hz per channel
Measurement range: 0–400°C
Accuracy: ±0.5°C (post-calibration)

Outcome:System ran continuously for 6 months with zero downtime. Process engineering used the data to re-tune all 12 zones. Defect rate dropped from 4.2% to 3.4%, saving ~$120K annually. Handed off maintenance docs and a web dashboard for operators.

Modular Assembly Fixture for PCB Testing

Developed a quick-change pogo-pin fixture that reduced board swap time from 45 seconds to 8 seconds, increasing test throughput by 3x and eliminating operator fatigue during high-volume runs.

Goal:Manual alignment of test probes was slow and error-prone. Operators had to visually align 32 pogo pins to pads while holding the board steady. Misalignment caused false failures and damaged connectors.

Approach:Designed a laser-cut acrylic jig with indexed locating pins matching the PCB mounting holes. Pogo pins press-fit into a 3D-printed carrier plate with spring-loaded retention. Hinged clamp arm locks the board in place with one motion.

Validation:Tested with 50 boards across 3 product variants. Measured cycle time, contact resistance, and operator feedback. Zero misalignments. Contact resistance stayed below 50mΩ across 500+ insertions.

First iteration used MDF for the base, but it warped after a week in the humid production area. Switched to 6mm acrylic with aluminum standoffs. Pogo pins sourced from Mill-Max with gold-plated tips for durability.

The clamp arm pivot used a simple piano hinge initially, but it flexed under clamping force. Upgraded to a stainless steel barrel hinge with a cam-lock mechanism. Added a foam gasket to prevent board flex during contact.

Alignment tolerance: ±0.2mm
Clamp force: 8–12 N (adjustable)
Cycle time: 8 sec (vs. 45 sec manual)

Outcome:Production adopted the fixture for all three board variants. Designed modular pin carriers so swapping between products takes 30 seconds (unscrew 4 bolts, swap carrier). Test throughput tripled. Operators reported significantly less hand strain. Documentation included CAD files, BOM, and assembly instructions for future variants.

Automated Vibration Test Rig Integration

Integrated a 6-axis IMU, data logger, and shaker controller into a unified test system for qualifying automotive sensors, cutting validation time from 2 days to 4 hours per unit.

Goal:Manual vibration testing required multiple disconnections, re-calibrations, and spreadsheet data entry. No real-time monitoring meant failures were only discovered post-test. Needed closed-loop automation with live pass/fail criteria.

Approach:Connected a Bosch BMI088 IMU to an Arduino Due via SPI, logging accelerometer and gyro data at 1 kHz. Integrated with the shaker's RS-232 interface to trigger sweep profiles and emergency stops. Built a LabVIEW dashboard for live FFT plots and threshold alerts.

Validation:Ran 10 qualification units through the full MIL-STD-810G profile. Detected two early failures (cracked solder joints) that would have passed manual inspection. All data auto-exported to timestamped CSV files for traceability.

Initial serial communication had timing issues—shaker commands occasionally dropped. Added a handshake protocol and retry logic. IMU mounting used double-sided foam tape at first, but resonance shifted the frequency response. Switched to a machined aluminum bracket with M3 screws.

LabVIEW code structured as a state machine: idle → configure → run → analyze → report. Added watchdog timer to auto-stop if accelerometer readings exceeded 20g (safety limit). Implemented rolling buffer to prevent memory overflow during long sweeps.

Sample rate: 1 kHz (IMU)
Frequency range: 10–2000 Hz
Pass threshold: ±5% deviation from nominal

Outcome:Test lab now runs 3 units per day instead of 1 every 2 days. Automated reporting reduced manual data entry errors to zero. Early failure detection saved rework costs on 2 production batches. Delivered a 60-page test procedure document, LabVIEW source code, and a spare IMU calibration kit.
